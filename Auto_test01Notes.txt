Lesson 9: Asynchronous Playwright API & Async/Await Foundation:Transition from Synchronous to Asynchronous automation.
Chúng ta sẽ đập bỏ hoàn toàn thư viện đồng bộ sync_playwright hiện tại để chuyển sang thế giới bất đồng bộ sử dụng async_playwright. 
Đây là nền tảng cốt lõi của lập trình hiệu năng cao trong Python giúp xử lý luồng không bị nghẽn (non-blocking).
    Bản chất Kỹ thuật: Synchronous vs. Asynchronous trong Automation
        Từ Lesson 1 đến Lesson 8, chúng ta đang dùng API đồng bộ (Synchronous - Sync):
        Cơ chế hoạt động: Khi script chạy đến lệnh page.goto(url) hoặc page.click(), 
        luồng xử lý của Python sẽ bị khóa cứng (blocked) lại. Python sẽ ngồi im ròng rã vài giây 
        để đợi trình duyệt tải xong trang web rồi mới chịu chạy tiếp dòng code tiếp theo.
        Nhược điểm: Cực kỳ lãng phí tài nguyên CPU và RAM, đồng thời khiến hệ thống không thể 
        chạy song song nhiều trình duyệt cùng một lúc (Parallel Testing).

        Hôm nay, chúng ta chuyển sang API bất đồng bộ (Asynchronous - Async):
        Cơ chế hoạt động: Sử dụng cặp từ khóa async và await. Khi gặp lệnh await page.goto(url), 
        Python sẽ phát lệnh cho trình duyệt chạy, nhưng thay vì ngồi đợi vô nghĩa, 
        Python sẽ lập tức giải phóng luồng (unblock) để đi xử lý các tác vụ khác. 
        Khi trình duyệt tải xong, nó sẽ báo lại và Python quay lại xử lý tiếp.
        Ưu điểm: Đây là viên gạch nền tảng bắt buộc để ở Lesson 10, 
        chúng ta có thể bật cùng lúc 5-10 trình duyệt chạy song song mà không làm sập RAM máy tính.
    Vì chúng ta chuyển sang cơ chế Async, cả 3 file chính trong hệ thống gồm login_page.py, conftest.py, và test_first_login.py 
    đều cần được nâng cấp cấu trúc đồng loạt.

    Vì chúng ta dùng kịch bản async/await, framework Pytest cần thêm một chiếc "chìa khóa" để dịch và chạy được code async. 
    Bạn chỉ cần cài thêm thư viện cầu nối pytest-asyncio: pip install pytest-asyncio
    Nếu không cài thêm sẽ báo lỗi khi chạy: 5 skipped, 5 warnings in 0.06s
        async def functions are not natively supported and have been skipped.
        You need to install a suitable plugin for your async framework, for example:
            - anyio
            - pytest-asyncio
            ...

    Actions),  bạn sẽ không thể ngồi nhìn màn hình terminal hay xem lệnh print() hiển thị trực tiếp được. Nếu test case bị lỗi giữa đêm,

Lesson 15: Hybrid Automation — Injecting API Tokens into UI Sessions

Trong các dự án Web Enterprise lớn, nếu hệ thống có hàng trăm UI test cases cần kiểm tra 
các tính năng sâu bên trong Dashboard (ví dụ: quản lý kế toán, cài đặt hệ thống, phân quyền...), việc test case nào cũng phải lặp lại các bước mở màn hình login, điền username, điền password rồi đợi trang web load sẽ gây ra sự lãng phí tài nguyên cực kỳ lớn và làm chậm tốc độ của CI/CD pipeline.
Hôm nay, chúng ta sẽ kết hợp sức mạnh của cả API lẫn UI để thực hiện kỹ thuật Hybrid Testing.

Luồng hoạt động ok:
1. Gửi một API Request đăng nhập ngầm để lấy mã token bảo mật về (chỉ mất vài mili-giây).
2. Khởi động một browser context trống thông qua Playwright UI.
3. Inject (Bơm trực tiếp) mã token này vào cookies hoặc localStorage của browser context đó.
4. Điều hướng thẳng trình duyệt đến URL /dashboard. Trình duyệt sẽ nhận diện token hợp lệ và cho phép vào thẳng 
trang quản trị mà hoàn toàn bỏ qua bước Login UI.
